### Archived 0.8.0 in-flight notes (for next-session reference)

**Bite plan**: 5 sub-bites of E (ADR design, fork implementation, zombie reaper, smoke test, closeout). Shipped 2026-05-23 from a single editing session.

**Design decision**: fork-per-accept (option F) over thread-per-accept (option T) because M2 only closes with a shared-state refactor under threading; fork closes M2 for free via address-space isolation. Single-thread epoll (option E) also rejected — would force every byte handler in `handle_client` to be yield-aware, doubling cognitive load on the line/IAC state machine. Single-track-with-arena (option S) only closes M1; M2 stays queued. Memory model: process-exit cleanup (option X) over per-conn freelist arena (option Y) — kernel-managed reclamation is strictly stronger than `fl_free`. Zombie reaper: non-blocking `sys_waitpid(-1, NULL, WNOHANG)` loop (option p) over `SIG_IGN` (option q) or `SA_NOCLDWAIT` (option r) — avoids the x86_64 sigaction trampoline trap that cyrius `lib/darshana.cyr` documents.

**Smoke verification**: 3 simultaneous TCP sessions via `/tmp/agora-concurrent-smoke.py` — all 3 got own banner + own anonymous response (proves no global collision, M2 closed). Reaper drains zombies from previous accept iterations (verified: round-1's 3 zombies cleared at round-2's accept). Steady-state: ≤ N zombies pending between accepts, where N = children that exited since last accept.

## Recent shipped

- **1.3.3** (2026-06-08) — **Jabberwacky** (Rollo Carpenter, 1988→), agora's first corpus-**learning** chatbot (`src/jabberwacky.cyr`; ADR 0015, chatbot lineage ADR 0012). A different engine kind from the fixed-script Eliza/PARRY: it **remembers what people say and replays it** — word-overlap retrieval over a `(stimulus, response)` corpus picks the closest prior stimulus and answers with the human reply that followed, then **learns** the transition (your previous line → your current line) so it can replay your words later. Corpus = a baked-in **seed** personality (~24 pairs, shared, never persisted) + a per-user **learned ring** (cap 64, oldest-learned evicts). Reuses ELIZA's text primitives, none of its reassembly engine (shares the chassis, not the engine). **Three surfaces:** `play jabberwacky` (ephemeral practice), `play jabberwacky solo` (login-gated; the learned layer **persists per-user across sessions** under `.users/<fp16>/games/jabberwacky.sav` via the QUEST stateful-door save path — agora's first persistent-state bot), and a private `/jabberwacky` couch. **No global corpus** — per-user isolation removes the cross-user poisoning / privacy / escape-injection surface ADR 0012 flagged; stored responses are ASCII-sanitised. No new deps. A pre-cut **5-dimension multi-agent adversarial review** (11 confirmed / 10 refuted, no critical/high) fixed a **medium** `jw_overlap` repeated-word double-count plus **low** save/load + matching robustness gaps (LF-stop on the tab-scan, honest save header, whole-word stimulus truncation, couch-reentry cursor reset, `you`/`your` stop-listed) and a **nit** load-loop cap. 166 → **178 tests** (t167-t178); 782,064 → **797,640 B**; pin 6.1.10 → **6.1.12**; smoke `15-jabberwacky.sh`. (A review agent's `git checkout` clobbered the uncommitted tests mid-run — restored; future review workflows run worktree-isolated.)
